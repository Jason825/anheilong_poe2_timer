# 项目协作说明

## 项目目标

本项目是《流放之路 2》剧情开荒透明悬浮计时器，产品名为“暗黑龙剧情计时器”，目标是帮助玩家在开荒时手动记录每个剧情节点耗时，并通过历史最快成绩优化路线。

第一版优先交付 Windows 可运行原型，不优先做安装包。

## 技术栈

- Electron：透明悬浮窗、置顶、点击穿透、托盘、全局快捷键。
- React + Vite：计时器界面、节点模板、历史成绩、PB 对比。
- 本地 JSON：保存设置、节点模板、历史记录和未完成记录。

## 常用命令

```bash
npm install
npm run dev
npm run build
```

## 关键目录

- `electron/`：Electron 主进程和 preload。
- `src/`：React 渲染进程。
- `docs/`：产品背景、阶段上下文和后续计划。

## 编码约定

- 所有用户可见按钮、菜单、设置项和提示文案默认使用中文。
- UI 保持透明、小巧、低遮挡，优先服务游戏实战。
- 应用启动后默认可点击/可拖动且背景不透明度 100% 可见。
- 托盘、窗口、任务栏和未来安装包/桌面快捷方式图标统一使用 `build/icon.ico`，图标源自 `public/brand/dark-dragon-logo.png`。
- 右上角控制按钮使用图标：迷你/列表/列表2、锁定、透明度、设置、最小化、关闭，不使用文字按钮。
- 用户点锁定后主面板不可移动且点击穿透，右上角按钮仍保持可见，但只有独立锁按钮可点击解锁。
- 迷你模式默认横向布局：当前章节在左，当前章节时间放大在右，总时间弱化放在下方；暂停时只改变时间颜色，不显示暂停文字。
- 计时面板和设置面板都不支持用户手动拖拽改变窗口大小。
- 迷你模式右上角第一个按钮用于切换迷你 / 列表模式 / 列表模式2；列表模式是在迷你模式基础上增加时间列表，列表模式2只展示关卡和当前时间。
- 列表模式的时间列表应显示在四个控制按钮下方。
- 设置必须作为独立 Electron 窗口打开，不能塞进主计时器窗口；设置窗口采用左侧图标栏 + 右侧内容区，模板页命名为“关卡模板”。
- 重置当前关卡、重置所有关卡、重置关卡模板等确认动作都使用自定义弹窗，不使用系统默认确认框。
- 快捷键设置只展示：开始/暂停、下一关、重置当前关卡、迷你/列表/列表2。
- 所有计时显示统一使用 `00:00:00` 格式。
- `透明度` 设置只作用于面板背景，不要影响文字、时间数字和图标。
- 快捷键应能在设置面板中修改并立即重新注册。
- 快捷键触发重置当前关卡时直接执行，不弹确认；鼠标点击重置当前关卡/重置所有关卡按钮时需要自定义确认弹窗。
- 历史记录第一条应展示“历史最佳记录”，由每个节点的历史最快时间组合生成。
- 历史记录支持保存本次、删除单条记录；设置窗口中使用下拉框选择历史记录。
- 关卡模板支持删除，删除模板会连带删除该模板创建的历史记录；计时中或暂停中不删除模板，确认弹层必须明确提示连带删除历史记录。
- 默认以手动快捷键记录为主，不做自动识别剧情进度。
- 保存关卡模板时只禁止与现有模板同名；允许相同节点结构使用不同模板名保存。历史最快仍按节点结构 `templateKey` 对比。
- 本地 JSON 需要兼容旧版本或异常数据；读取时必须做防御性归一化，避免坏模板、坏历史记录、坏快捷键或坏 activeRun 导致渲染崩溃。

## 安全边界

- 不读取游戏内存。
- 不注入游戏进程。
- 不自动识别游戏画面。
- 不自动发送键鼠操作给游戏。
- 不记录账号密码、Token、支付信息或其他敏感数据。
